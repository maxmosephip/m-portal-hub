# m-portal-hub

## 项目简介

**m-portal-hub** 是一个用于归档和发布多个独立 HTML 页面的静态仓库。  
本仓库不针对任何特定域名或网站，仅作为不同 HTML 页面资源的统一托管与索引入口。

所有页面均为独立的静态文件，便于快速部署、预览及分享。

## 目录结构

```
m-portal-hub/
├── pages/          # 存放所有独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/         # 公共资源（样式、脚本、图片等）
├── index.html      # 仓库入口页面（可选）
└── README.md       # 本文件
```

- `pages/` 目录下的每个 HTML 文件对应一个独立页面。
- `assets/` 目录用于存放多个页面可能共用的静态资源，如 CSS、JavaScript、图标等。

## 页面归档说明

- 每个 HTML 页面均为自包含的独立文件，可单独访问。
- 页面内容遵循原始归档时的状态，不进行额外修改或优化。
- 若页面依赖外部资源（如 CDN 库），请确保资源链接有效或自行托管至 `assets/` 目录。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/m-portal-hub.git
   ```
2. 直接通过浏览器打开 `pages/` 下的任意 HTML 文件，或部署至任意静态托管服务。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并更新 `index.html` 中的导航链接（如有）。
- 更新页面：直接替换对应文件，注意保持文件名不变以避免外部引用失效。
- 删除页面：移除 `pages/` 中对应文件，并同步更新索引。

> 本仓库主要用于个人或团队内部归档与测试，不保证长期在线或兼容性。
